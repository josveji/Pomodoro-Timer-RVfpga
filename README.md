# Pomodoro-Timer-RVfpga

**Authors:**

* Josué M. Jiménez Ramírez
* Gabriel Vega Chavez

## Project Description

This project implements a **Pomodoro Timer** on an FPGA using the SweRVolf RISC-V core on a Nexys A7 board. The Pomodoro Timer is a time management technique that divides work into 25-minute intervals (work sessions) followed by short 5-minute breaks, aiming to improve productivity and reduce mental fatigue.

### Main Features

* **Customizable Settings:** Allows adjusting work time (1–60 minutes) and break time (1–20 minutes) using buttons.
* **Timer States:**

  * **Configuration (CONFIG):** Time adjustment.
  * **Work (WORK):** Countdown of work time.
  * **Break (BREAK):** Countdown of break time.
* **User Interface:**

  * **Buttons:** Center (start), Up/Down (adjust work time), Left/Right (adjust break time).
  * **7-Segment Display:** Shows remaining time in MM:SS format and the current state.
  * **LEDs:** Visual indicators for different states (configuration, work, break).
  * **Audio Alarm:** Audible signal when a work or break session ends.
* **Interrupts:** Uses hardware interrupts for the timer and buttons, ensuring accurate and efficient response.

## How It Was Achieved

### System Architecture

The project is based on the **RVfpga** (RISC-V FPGA) design, which integrates a RISC-V processor (SweRVolf core) with memory-mapped I/O peripherals. Modifications were made to the original design to adapt it to the Nexys A7 board, including support for LEDs, buttons, a 7-segment display, and audio output.

### Hardware Components

* **FPGA:** Nexys A7 (with SweRVolf RISC-V core).
* **Peripherals:**

  * GPIO for LEDs and switches.
  * Push buttons.
  * 7-segment display.
  * Precision timer (PTC - Programmable Timer Counter).
  * Audio output for alarm.
* **Modified Files:**

  * `rvfpganexys.sv`: FPGA toplevel module adapted for the Nexys A7 board.
  * `rvfpganexys.xdc`: FPGA design constraints.
  * `swervolf_core.v`: SweRVolf RISC-V core.
  * `wb_intercon.v` and `wb_intercon.vh`: Wishbone interconnection for module communication.

### Software

* **Language:** C, running on the RISC-V processor.
* **Framework:** Western Digital RISC-V SDK, integrated with PlatformIO.
* **Key Functionalities:**

  * **State Management:** State machine logic for configuration, work, and break modes.
  * **Interrupts:** ISR for the timer (countdown every second) and buttons (configuration).
  * **Display and LEDs:** Real-time updates of the 7-segment display and LEDs according to the current state.
  * **Alarm:** Generation of an audible tone using PWM on the audio output.
* **Compilation:** Uses PlatformIO to build the firmware and generate the FPGA bitstream.

### Development Process

1. **Hardware Design:** Modification of RVfpga Verilog files to integrate the required peripherals on the Nexys A7 board.
2. **Software Implementation:** Development of C code handling timer logic, interrupts, and I/O.
3. **Testing:** Test code in the `Testing_codes/` folder to validate individual components (e.g., `POMODORO.c`, `proyecto_final.c`, `prueba_ptc.c`).
4. **Synthesis and Programming:** Generation of the `rvfpganexys_buttons_and_aux.bit` bitstream to program the FPGA.
5. **Integration:** Use of PlatformIO to compile and upload the firmware to the embedded system.

### Repository Structure

* `Pomodoro-Timer-RVfpga/`: Main project with source code and configuration.

  * `src/`: Timer C source code.
  * `platformio.ini`: PlatformIO configuration.
  * `.pio/`: Files generated by PlatformIO.
* `RVfga_modified_files/`: Modified Verilog files for the FPGA.
* `Testing_codes/`: Test code and previous versions.
* `rvfpganexys_buttons_and_aux.bit`: Synthesized bitstream for the FPGA.

### Requirements

* **Hardware:** Nexys A7 FPGA board.
* **Software:** PlatformIO with support for chipsalliance/swervolf_nexys.
* **Dependencies:** Western Digital RISC-V SDK.

### Usage Instructions

1. Program the FPGA with the provided bitstream.
2. Compile and upload the firmware using PlatformIO: `pio run -t upload`.
3. On the board:

   * Use the buttons to configure times.
   * Press the center button to start the timer.
   * The display shows the remaining time and LEDs indicate the current state.
   * When finished, the alarm sounds and the state changes automatically.

This project demonstrates the integration of embedded software with reconfigurable hardware, applying concepts from digital systems, interrupts, and time management in a practical productivity context.
